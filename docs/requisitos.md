# Requisitos do Projeto

## Requisitos Funcionais (RF)
- **RF1**: Usuário deve poder visualizar dashboard de dados de CO₂.
- **RF2**: Sistema deve permitir login e autenticação.
- **RF3**: Usuário pode exportar dados em CSV.
- **RF4**: Administrador pode atualizar os dados do dashboard.

## Requisitos Não Funcionais (RNF)
- **RNF1**: Dashboard deve carregar em menos de 3 segundos.
- **RNF2**: Aplicação responsiva para desktop e mobile.
- **RNF3**: Dados devem ser armazenados de forma segura.
- **RNF4**: Interface deve ser intuitiva e amigável.
